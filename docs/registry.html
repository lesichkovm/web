Registry

The Registry provides a simple way to store and retrieve data. It uses @lesichkovm/registryjs under the hood to provide persistent key-value storage that survives page reloads.

$$.set(key, value)

Stores a value in the registry. The value is automatically serialized and persisted.

// Set a simple value
$$.set('theme', 'dark');

// Set a complex object
$$.set('user.preferences', {
  fontSize: 14,
  showNotifications: true
});

$$.get(key)

Retrieves a value from the registry. If the key doesn't exist, it returns null.

const theme = $$.get('theme'); // 'dark'
const prefs = $$.get('user.preferences'); // { fontSize: 14, ... }

How it works

The Registry is initialized when WebJS loads. It automatically detects the best storage mechanism (usually localStorage) to ensure that your data persists across sessions and page refreshes. This makes it ideal for storing user preferences, session state, and cached data.
