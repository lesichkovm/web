# WebJS Overview

## Project Purpose

WebJS is a lightweight framework for building multipage web applications, exposing its utilities through a `$$` global to streamline common client-side tasks.@README.md#1-18

## Key Features

- URL helpers for reading, composing, and navigating with query parameters, enabling simple in-app routing flows.@README.md#13-118
- Authentication utilities for persisting tokens and user objects through the shared registry.@README.md#121-137
- Publish/subscribe event system for decoupled communication between widgets and pages.@README.md#139-181
- Dynamic widget loader that fetches markup, injects it into the DOM, and safely executes embedded scripts.@README.md#48-80 @src/loadWidgets.js#1-45
- Background image loader that lazily swaps in assets once they finish downloading, removing temporary spinners or placeholders.@src/loadBackgroundImages.js#1-69

## Architecture Overview

### Core Runtime

The `Initialize` class wires together the runtime, constructing a project-scoped registry, navigation helper, pub/sub bus, and widget loader while exposing higher-level convenience methods via the `$$` global entry point.@src/web.js#1-200
RegistryJS provides the persistent key-value store that backs most runtime state and is instantiated per host using the project-specific application identifier.@src/web.js#23-123 @src/core/utils.js#17-25

### Core Modules

- `core/utils` delivers string trimming helpers and unique ID generation aligned with the configurable application identifier.@src/core/utils.js#1-25
- `core/url` centralizes URL introspection and API/base URL derivation with sensible defaults for local development.@src/core/url.js#1-94
- `core/navigation` wraps browser navigation with optional query parameter support and `_blank` handling tailored for tests and production.@src/core/navigation.js#1-75
- `core/auth` stores and retrieves user, token, and language data from the registry, defaulting language fallbacks to English.@src/core/auth.js#1-60
- `core/pubsub` implements a lightweight token-based publish/subscribe bus used by the runtime and widgets.@src/core/pubsub.js#1-68

### Widget & Asset Helpers

- `loadWidgets` scans for elements annotated with `data-widget-url`, fetches the referenced HTML, injects it, and executes any inline scripts safely.@src/loadWidgets.js#1-45
- `loadBackgroundImages` locates `div[data-background-image]` elements, preloads the referenced image, then swaps it into the background while removing interim loaders.@src/loadBackgroundImages.js#1-69

## Build & Distribution

The project bundles `src/web.js` with esbuild into `dist/web.bundle.js`, applies a versioned banner, targets modern browsers, and outputs an IIFE named `WebJS` ready for CDN distribution.@src/esbuild.config.js#1-45
Build tooling is orchestrated through npm scripts for preparing the `dist` directory, bundling, and minifying the distribution artifact.@package.json#7-26

## Usage Patterns

When included via CDN or bundled into an application, WebJS exposes its API through `window.$$` or `window.WebJS`, allowing direct access to URL, auth, navigation, and registry helpers from page scripts or modules.@README.md#40-191

## Development & Testing

The repository uses Jest with a JSDOM test environment and project-specific setup file, runnable via `npm test` for unit and integration coverage.@package.json#7-26 @jest.config.js#1-8

- `src/` — core runtime, utilities, and helper modules consumed by the bundled build.@src/web.js#1-200
- `dist/` — compiled bundles generated by the build pipeline.
